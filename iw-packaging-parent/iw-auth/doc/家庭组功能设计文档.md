# 家庭组功能设计文档

## 一、功能概述

### 1.1 设计目标
将现有的 ToC 单用户体系扩展为支持家庭组的多用户协作体系，允许用户创建或加入家庭组，实现数据共享和协作管理。
将现有的业务功能模块从单用户ToC数据权限扩展为家庭组多用户数据权限，用户可以看到同一家庭组下所有用户的共享数据。

### 1.2 家庭组核心功能
- 用户可以创建家庭组（成为组长）
- 用户可以通过邀请码加入家庭组
- 家庭组成员可以共享查看特定数据
- 用户只能加入到一个家庭组，加入到新家庭组前必须先退出原家庭组
- 家庭组角色除了群主、成员外，额外增加家长、儿童两种类型
  - 家长角色也可以邀请成员，但没有修改家庭组信息、转让群主、解散家庭组功能
  - 儿童角色也是普通成员，但是儿童角色的业务数据默认只能是共享的，不能手动切换
- 家庭组角色可以分配角色，群主可以分配所有角色，家长可以给成员和儿童分配成员或儿童角色，其他角色无权限

### 1.3 业务共享数据核心功能
* 用户查询支持查看同一家庭组下的所有共享数据。
* 业务查询共享数据的操作做一个统一封装，代码侵入性不要太强，避免后期每多加一个业务都得写一遍。
* 共享数据是每个家庭组成员自由决定是否可以共享的。
* 共享开关作为家庭组成员的默认值，业务数据也可以再手动切换是否共享。
* 默认共享开关默认关闭，儿童角色默认开启且不可手动修改。
* 角色从非儿童切换为儿童时，默认共享开关自动改为开启；儿童切换为非儿童时开关保持原值。

## 二、数据模型设计

### 2.1 核心表结构

#### 2.1.1 家庭组表 (auth_family_group)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int unsigned | 主键 |
| group_name | varchar(64) | 家庭组名称 |
| group_avatar | varchar(255) | 家庭组头像 |
| invite_code | char(8) | 邀请码（唯一） |
| owner_user_id | int unsigned | 组长用户ID |
| member_count | tinyint unsigned | 当前成员数量 |
| max_member | tinyint unsigned | 最大成员数（默认10） |
| group_desc | varchar(255) | 家庭组描述 |
| status | tinyint(1) | 状态（1启用 2禁用） |
| deleted | tinyint(1) | 逻辑删除标记 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |

**业务规则：**
- 邀请码为8位字符，由大写字母和数字组成，排除易混淆字符（0,O,1,I,L）
- 创建家庭组时自动生成唯一邀请码
- 组长不能退出家庭组，只能解散或转让
- 成员数量达到上限时不能再邀请新成员

#### 2.1.2 家庭组成员表 (auth_family_member)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int unsigned | 主键 |
| group_id | int unsigned | 家庭组ID |
| user_id | int unsigned | 用户ID |
| member_role | tinyint(1) | 成员角色（1组长 2家长 3成员 4儿童） |
| default_shared | tinyint(1) | 默认共享开关（0关闭 1开启） |
| member_nickname | varchar(32) | 成员昵称（在家庭组内的昵称） |
| join_time | datetime | 加入时间 |
| status | tinyint(1) | 状态（1正常 2已退出 3已被移除） |
| deleted | tinyint(1) | 逻辑删除标记 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |

**业务规则：**
- group_id + user_id 唯一索引，一个用户在同一家庭组只能有一条记录
- 用户同一时刻只能加入一个家庭组，加入新组前必须先退出当前组
- member_nickname 默认使用用户的 name，可以自定义修改
- default_shared 默认值为0；儿童角色必须为1且不可手动修改

#### 2.1.3 家庭组邀请记录表 (auth_family_invite)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int unsigned | 主键 |
| group_id | int unsigned | 家庭组ID |
| invite_user_id | int unsigned | 邀请人用户ID |
| invited_user_id | int unsigned | 被邀请人用户ID（可为空） |
| invite_code | char(8) | 邀请码 |
| invite_status | tinyint(1) | 邀请状态（1待接受 2已接受 3已拒绝 4已过期） |
| expire_time | datetime | 过期时间 |
| accept_time | datetime | 接受时间 |
| deleted | tinyint(1) | 逻辑删除标记 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |

**业务规则：**
- 记录每次邀请的详细信息
- 邀请码可以重复使用（同一家庭组的邀请码相同）
- 支持邀请过期机制（可选）

#### 2.1.4 用户表扩展 (auth_user)

新增字段：
- `current_group_id` (int unsigned): 当前激活的家庭组ID，0表示个人模式

**业务规则：**
- 用户可以在个人模式和家庭组模式之间切换
- 切换家庭组时，更新此字段
- 查询数据时，根据此字段判断是查询个人数据还是家庭组数据

### 2.2 业务表改造

所有需要支持家庭组共享的业务表，统一增加以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| group_id | int unsigned | 家庭组ID（0表示个人数据） |
| is_shared | tinyint(1) | 是否共享（0否 1是） |

**已改造的业务表：**
1. bookkeeping_records（记账记录）
2. bookkeeping_actions（记账行为）
3. bookkeeping_budget（记账预算）
4. eat_meal（用餐记录）
5. eat_fridge_food（冰箱食材）
6. points_task_basics（任务基础）
7. points_task_group（任务分组）
8. points_task_plan（任务计划）
9. base_application_account（应用账号）

**特殊字段：**
- `points_task_basics` 表额外增加 `assigned_user_id` 字段，用于任务指派功能

## 三、核心业务流程

### 3.1 创建家庭组流程

```
1. 用户填写家庭组名称、描述等信息
2. 系统生成唯一的8位邀请码
3. 创建家庭组记录（owner_user_id = 当前用户ID）
4. 创建家庭组成员记录（member_role = 1 组长）
5. 更新用户的 current_group_id
6. 返回家庭组信息和邀请码
```

**接口设计：**
```java
POST /auth-service/family-group/create
Request:
{
    "groupName": "我的家庭",
    "groupDesc": "温馨的家",
    "groupAvatar": "https://..."
}

Response:
{
    "code": 200,
    "data": {
        "groupId": 1,
        "groupName": "我的家庭",
        "inviteCode": "ABC12345",
        "memberCount": 1
    }
}
```

### 3.2 加入家庭组流程

```
1. 用户输入邀请码
2. 系统验证邀请码是否有效
3. 检查家庭组是否已满员
4. 检查用户是否已加入其他家庭组（若已加入，提示先退出当前组）
5. 创建家庭组成员记录（member_role = 3 成员）
6. 更新家庭组的 member_count
7. 创建邀请记录（invite_status = 2 已接受）
8. 更新用户的 current_group_id
9. 返回加入成功信息
```

**接口设计：**
```java
POST /auth-service/family-group/join
Request:
{
    "inviteCode": "ABC12345"
}

Response:
{
    "code": 200,
    "data": {
        "groupId": 1,
        "groupName": "我的家庭",
        "memberCount": 2,
        "joinTime": "2024-03-09 14:30:00"
    }
}
```

### 3.3 切换家庭组流程

```
1. 用户选择切换到个人模式或当前家庭组模式
2. 验证用户是否是该家庭组成员
3. 更新用户的 current_group_id
4. 返回切换成功信息
```

**接口设计：**
```java
POST /auth-service/family-group/switch
Request:
{
    "groupId": 1  // 0表示切换到个人模式
}

Response:
{
    "code": 200,
    "data": {
        "currentGroupId": 1,
        "groupName": "我的家庭"
    }
}
```

### 3.4 退出家庭组流程

```
1. 验证用户不是组长（组长不能退出，只能解散）
2. 更新家庭组成员状态为"已退出"
3. 更新家庭组的 member_count
4. 如果用户当前激活的是该家庭组，则切换到个人模式
5. 返回退出成功信息
```

### 3.5 解散家庭组流程（仅组长）

```
1. 验证用户是组长
2. 将所有成员状态更新为"已被移除"
3. 将家庭组状态更新为"禁用"
4. 所有成员的 current_group_id 如果是该家庭组，则重置为0
5. 返回解散成功信息
```

## 四、数据权限控制

### 4.1 查询数据规则

**个人模式（current_group_id = 0）：**
```sql
WHERE user_id = #{currentUserId} AND group_id = 0
```

**家庭组模式（current_group_id > 0）：**
```sql
WHERE group_id = #{currentGroupId} AND is_shared = 1
```

**组合查询（个人数据 + 家庭组共享数据）：**
```sql
WHERE (user_id = #{currentUserId} AND group_id = 0)
   OR (group_id = #{currentGroupId} AND is_shared = 1)
```

### 4.2 创建数据规则

**个人模式：**
- group_id = 0
- is_shared = 0
- user_id = 当前用户ID

**家庭组模式：**
- group_id = 当前家庭组ID
- is_shared = 用户选择（默认读取家庭组成员 default_shared，默认0；儿童角色强制默认1）
- user_id = 当前用户ID

### 4.3 修改/删除数据规则

**权限验证：**
1. 个人数据：只能修改/删除自己创建的数据（user_id = 当前用户ID）
2. 家庭组数据：
   - 组长可以修改/删除所有家庭组数据
   - 成员只能修改/删除自己创建的数据

## 六、技术实现要点

### 6.1 邀请码生成算法

```java
/**
 * 生成8位邀请码
 * 规则：大写字母+数字，排除易混淆字符(0,O,1,I,L)
 */
public String generateInviteCode() {
    String chars = "ABCDEFGHJKMNPQRSTUVWXYZ23456789";
    Random random = new Random();
    StringBuilder code = new StringBuilder();
    for (int i = 0; i < 8; i++) {
        code.append(chars.charAt(random.nextInt(chars.length())));
    }
    // 检查唯一性，如果重复则重新生成
    return code.toString();
}
```

### 6.3 用户上下文扩展

```java
/**
 * 扩展用户上下文，增加家庭组信息
 */
public class UserContext {
    private Integer userId;
    private Integer currentGroupId;  // 新增
    private String groupName;        // 新增
    private Integer memberRole;      // 新增

    public boolean isPersonalMode() {
        return currentGroupId == null || currentGroupId == 0;
    }

    }
```

## 九、注意事项

### 9.1 数据安全
- 家庭组数据只能被组内成员访问
- 退出家庭组后不能再访问该家庭组数据
- 敏感数据（如密码）不建议共享

### 9.2 性能优化
- group_id 和 user_id 建立联合索引
- 家庭组成员列表使用缓存
- 大数据量查询时注意分页

### 9.3 用户体验
- 首次使用时引导用户创建或加入家庭组
- 提供快速切换模式的入口
- 明确标识数据的归属和共享状态
